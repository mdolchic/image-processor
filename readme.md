# image_processor

`image_processor` — консольное приложение на C++ для обработки 24-битных BMP-изображений без сторонних библиотек для работы с графикой. Проект реализует полный цикл обработки: чтение BMP-файла, построение цепочки фильтров, последовательное преобразование изображения и сохранение результата.

## О проекте

Программа работает с несжатым `24-bit BMP` и построена как набор независимых компонентов, каждый из которых отвечает за отдельный этап обработки изображения. Такая структура упрощает сопровождение проекта и позволяет безболезненно расширять его новыми фильтрами.

В проекте реализованы:

- чтение BMP с `BITMAPINFOHEADER`
- запись 24-битного BMP с учетом выравнивания строк
- разбор аргументов командной строки
- последовательный пайплайн фильтров
- инфраструктура для добавления новых эффектов

## Поддерживаемые фильтры

- `-crop width height` — обрезка изображения по верхнему левому углу
- `-gs` — перевод в оттенки серого
- `-neg` — негатив
- `-sharp` — повышение резкости
- `-edge threshold` — выделение границ по порогу
- `-blur sigma` — гауссово размытие
- `-emboss` — фильтр тиснения

## Архитектура

Проект разделен на несколько ключевых частей.

- `image_processor.cpp` — точка входа и координация всей обработки
- `CLAParser` — разбор аргументов командной строки и формирование описаний фильтров
- `BMPImage` — модель изображения и контейнер для пикселей
- `BMPReader` — чтение BMP-файла и преобразование бинарных данных в `BMPImage`
- `BMPWriter` — обратная запись `BMPImage` в BMP
- `FilterFactory` — создание конкретных объектов фильтров по CLI-дескрипторам
- `Pipeline` — последовательное применение фильтров
- `filters/*` — базовые классы и реализации эффектов

## Устройство обработки

Обработка изображения проходит по фиксированной схеме:

1. парсинг входного и выходного пути, а также списка фильтров
2. чтение BMP-файла в объект `BMPImage`
3. создание объектов фильтров через `FilterFactory`
4. последовательное применение фильтров через `Pipeline`
5. сохранение результата через `BMPWriter`

## Особенности реализации

- пиксель хранится в формате `BGR`, что соответствует внутреннему представлению BMP
- для фильтров, работающих с соседними пикселями, используется копия исходного изображения
- для краевых пикселей в сверточных фильтрах предусмотрена отдельная обработка границ
- проект не использует сторонние библиотеки для обработки изображений
